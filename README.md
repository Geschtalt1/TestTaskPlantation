Проект модульной системы теплицы для **Unreal Engine 5.7**!

---

## 📦 1. Как устроены данные

Данные в системе разделены на **конфигурации** (ассетные данные) и **состояние** (данные в рантайме).

- **📜 Конфигурации (Assets):**
  Все параметры растений (скорость роста, требуемые ресурсы, названия) хранятся в `UPlantData`. Это `Primary Data Asset`.
  _Такой подход позволяет настраивать баланс без программиста: каждый тип растения — это отдельный ассет._

- **📊 Состояние (Runtime Data):**
  Состояние каждой конкретной ячейки хранится в структуре `FPlantCell`.
  _Здесь находится текущий прогресс роста, ссылки на ассеты и актуальные значения ресурсов (вода, свет)._

## 🧠 2. Где находится основная логика

Логика распределена по компонентам для разделения зон ответственности:

- **🧠 `UGreenhouseComponent`:**
  Это «мозг» системы. Он управляет массивом ячеек, отвечает за симуляцию (`UpdateSimulation`), поиск по координатам и содержит основную логику.
  _Здесь считается рост, потребление ресурсов и проверяются условия._

- **⏱️ `UGameTimeComponent`:**
  Глобальный компонент, управляющий временем в игре.
  _Он оповещает теплицу о тиках симуляции, чтобы та могла обновиться._

- **🛠️ `UGreenhouseLibrary`:**
  Статический класс с вспомогательными функциями.
  _Сюда выносится общая математика, чтобы не дублировать код._

## 🏗️ 3. Почему выбрана такая архитектура

1.  **📊 Гибкость (Data-Driven):** Использование `UPlantData` позволяет легко настраивать баланс через редактор.
2.  **⚙️ Разделение ответственности:** Компонент управляет, данные хранят настройки, структура хранит состояние.
3.  **⚡ Производительность:** Логика инкапсулирована и использует эффективные структуры данных.
4.  **🚀 Масштабируемость:** Легко добавлять новые механики, не ломая старую систему.

## 🛠️ 4. Что можно улучшить дальше

Вот идеи для развития системы:

- **⚡ Оптимизация поиска ресурсов:**
  Сейчас ресурсы в `FPlantGrowthResourceConfig` хранятся в массиве (`TArray`). Поиск нужного ресурса (например, воды) требует перебора.
  _➡️ **Решение:** Заменить массив на `TMap<FGameplayTag, ...>`, чтобы доступ к ресурсу был мгновенным._
